Vireos Provider Credentials Setup Runbook

Prepared: March 10, 2026

This runbook covers the provider credentials required by the Vireos backend and exactly where to place them in backend/.env.

1) Before You Start

  1. Open backend/.env.
  2. Decide callback base URL:
    • Local: http://localhost:13000
    • Public testing: https://<your-public-url>
  3. If using public URL, update API_BASE_URL, LINKEDIN_REDIRECT_URI, and FACEBOOK_REDIRECT_URI.

2) OpenAI

Env Vars

  • OPENAI_API_KEY
  • OPENAI_MODEL (optional)
  • OPENAI_ORG_ID (optional)

Steps

  1. Sign in to OpenAI Platform and open API keys.
  2. Create a new secret key and copy it immediately.
  3. Set OPENAI_API_KEY in backend/.env.
  4. Set model values:
    • OPENAI_MODEL=gpt-5.2-mini
    • OPENAI_AGENT_MODEL=gpt-5.2

3) LinkedIn OAuth

Env Vars

  • LINKEDIN_CLIENT_ID
  • LINKEDIN_CLIENT_SECRET
  • LINKEDIN_REDIRECT_URI

Steps

  1. Sign in to LinkedIn Developer portal and create/open your app.
  2. Configure OAuth redirect URI:
    • http://localhost:13000/api/v1/oauth/linkedin/callback, or
    • https://<your-public-url>/api/v1/oauth/linkedin/callback.
  3. Copy Client ID and Client Secret.
  4. Set them in backend/.env, and ensure redirect URI is an exact match.

4) Facebook / Meta OAuth

Env Vars

  • FACEBOOK_APP_ID
  • FACEBOOK_APP_SECRET
  • FACEBOOK_REDIRECT_URI
  • FACEBOOK_WEBHOOK_VERIFY_TOKEN

Steps

  1. Sign in to Meta for Developers and create/open your app.
  2. Add Facebook Login and configure Valid OAuth Redirect URI.
  3. Use:
    • http://localhost:13000/api/v1/oauth/facebook/callback, or
    • https://<your-public-url>/api/v1/oauth/facebook/callback.
  4. Copy App ID and App Secret into backend/.env.
  5. Set FACEBOOK_WEBHOOK_VERIFY_TOKEN to a long random string.

5) Stripe

Env Vars

  • STRIPE_SECRET_KEY
  • STRIPE_PUBLISHABLE_KEY
  • STRIPE_WEBHOOK_SECRET

Steps

  1. Sign in to Stripe and choose Test or Live mode.
  2. Copy secret and publishable keys from API keys page.
  3. Create webhook endpoint:
    • https://<your-public-url>/api/v1/webhooks/stripe
    • For localhost, use Stripe CLI forwarding.
  4. Copy endpoint signing secret (whsec_...) to STRIPE_WEBHOOK_SECRET.

6) SendGrid

Env Vars

  • SENDGRID_API_KEY
  • SENDGRID_FROM_EMAIL
  • SENDGRID_FROM_NAME

Steps

  1. Sign in to SendGrid.
  2. Verify sender identity (single sender or domain auth).
  3. Create API key with Mail Send permission.
  4. Set API key and verified sender values in backend/.env.

7) AWS S3

Env Vars

  • AWS_ACCESS_KEY_ID
  • AWS_SECRET_ACCESS_KEY
  • AWS_S3_BUCKET
  • AWS_REGION
  • AWS_S3_ENDPOINT (optional)

Steps

  1. Create/select an S3 bucket for Vireos uploads.
  2. Configure bucket CORS for your frontend origins.
  3. Create least-privilege IAM policy for this bucket.
  4. Create IAM user/access key and set all values in backend/.env.

8) Apply and Test Credentials

  1. Restart backend:
    • docker compose -f backend/docker-compose.yml down
    • docker compose -f backend/docker-compose.yml up -d --build
  2. Check readiness:
    • curl -H 'X-Forwarded-Proto: https' http://localhost:13000/health/ready
  3. Run one real integration test for each provider feature path.

9) Security Rules

  1. Never commit real credentials to git.
  2. Use separate test and production credentials.
  3. Use least privilege for all provider keys.
  4. Rotate any credential exposed in logs/screenshots.
